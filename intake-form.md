# First 7 Days Execution Plan

Use this plan to start selling Codebase Tune-Up this week.

## Day 1: Package The Offer

- Publish the README as the public service page.
- Publish the pricing page.
- Prepare the intake form.
- Prepare the report templates.
- Decide which package you want to sell first.

Suggested focus: start with the 299 euro Tune-Up + PR because it produces a concrete deliverable.

## Day 2: Create A Sample

- Run the workflow on one repository you own.
- Create a sample report using the English or Spanish template.
- Create one small PR or mock PR description.
- Save before/after examples without exposing private code.

Goal: have proof that the service produces practical output.

## Day 3: Build A Lead List

- List 20 relevant prospects.
- Include open-source maintainers, SaaS founders, agencies, freelance developers, and local businesses.
- Personalize each message.
- Do not use fear-based claims or fake urgency.

Goal: send only relevant, permission-based outreach.

## Day 4: Send Sample Offers

- Send personalized outreach messages.
- Offer a free sample review for public repositories.
- Ask for explicit authorization before reviewing private repositories.
- Track replies in a simple spreadsheet.

Goal: book 2-3 sample reviews.

## Day 5: Deliver First Samples

- Complete the free sample reviews.
- Keep each sample short: 1-2 findings.
- Use concrete file evidence.
- Offer the 99 euro Mini Tune-Up or 299 euro Tune-Up + PR as the next step.

Goal: convert one sample into a paid review.

## Day 6: Complete First Paid Review

- Confirm authorization and scope.
- Review repository contents only.
- Produce the top 5 findings.
- Recommend one minimal PR.
- If the package includes it, create the PR with verification and rollback notes.

Goal: deliver a professional report quickly.

## Day 7: Turn Delivery Into A Repeatable System

- Save the finished report as a reusable example.
- Improve the prompts based on what worked.
- Add the GitHub Actions workflow skeleton to your service repo.
- Write a short case study without exposing private code.
- Offer monthly maintenance to the first satisfied client.

Goal: move from one-off work to a repeatable service.
